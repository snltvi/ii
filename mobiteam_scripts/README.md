# Инструкция по использованию скрипта Fort Monitor Odometer

## Описание
Скрипт автоматически получает показания одометра транспортного средства при выезде из гаража и возвращении в гараж через API Fort Monitor 3.

## Требования

### Python библиотеки
```bash
pip install requests
```

## Настройка

### 1. Откройте файл `fort_monitor_odometer.py`

### 2. Найдите секцию настроек (строки ~280):
```python
BASE_URL = "https://web.fort-monitor.ru"  # Или адрес вашего сервера
USERNAME = "ваш_логин"
PASSWORD = "ваш_пароль"
```

### 3. Замените на свои данные:
- **BASE_URL** - адрес вашего сервера Fort Monitor
  - Для облачного хостинга: `https://web.fort-monitor.ru`
  - Для собственного сервера: `http://ваш-сервер:порт`
  
- **USERNAME** - ваш логин в Fort Monitor
- **PASSWORD** - ваш пароль

## Использование

### Запуск скрипта:
```bash
python fort_monitor_odometer.py
```

### Пошаговая работа:

1. **Скрипт подключится к API** и выведет список транспортных средств
   ```
   Доступные транспортные средства:
     1. ID: 123 - КамАЗ-5320 (А123АА77)
     2. ID: 124 - ГАЗель-3302 (В456ВВ77)
   ```

2. **Выберите транспортное средство** (введите номер)
   ```
   Выберите номер транспортного средства: 1
   ```

3. **Скрипт покажет список геозон**
   ```
   Доступные геозоны:
     1. ID: 10 - Главный гараж
     2. ID: 11 - Склад №1
     3. ID: 12 - База
   ```

4. **Выберите геозону гаража** (введите номер)
   ```
   Выберите номер геозоны гаража: 1
   ```

5. **Введите дату** (или нажмите Enter для сегодняшнего дня)
   ```
   Введите дату (ДД.ММ.ГГГГ) или нажмите Enter для сегодняшнего дня: 05.02.2025
   ```

## Что делает скрипт

### 1. Подключение к API
- Авторизуется в системе Fort Monitor
- Получает SessionId для последующих запросов

### 2. Получение данных
- Загружает список транспортных средств
- Загружает список геозон
- Запрашивает трек движения за выбранный день

### 3. Анализ геозоны
- Проходит по всем точкам трека
- Определяет, когда транспорт вышел из геозоны (выезд)
- Определяет, когда транспорт вернулся в геозону (въезд)
- Фиксирует показания одометра в эти моменты

### 4. Расчет пробега
- Вычисляет разницу между показаниями одометра при въезде и выезде

## Ограничения текущей версии

1. **Геозоны**: Скрипт работает только с **круговыми геозонами**
   - Для геозон-полигонов нужна доработка функции проверки точки

2. **Одна смена в день**: Скрипт находит только первый выезд и последний въезд
   - Если было несколько смен, учитывается только первая и последняя поездка

## Возможные ошибки

### Ошибка подключения:
```
✗ Ошибка подключения: 401
```
**Решение**: Проверьте логин и пароль

### SessionId не получен:
```
✗ Ошибка: SessionId не получен
```
**Решение**: Проверьте версию Fort Monitor API на сервере

### Трек движения не найден:
```
⚠ Трек движения не найден
```
**Решение**: 
- Проверьте, была ли активность транспорта в выбранный день
- Проверьте, корректно ли работает GPS терминал
