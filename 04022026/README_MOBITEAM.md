# Инструкция по использованию скрипта Mobiteam GPS API

## Описание
Скрипт автоматически получает показания одометра транспортного средства при выезде из гаража и возвращении через API системы мониторинга Mobiteam.

## Требования

### Python библиотеки
```bash
pip install requests
```

## Получение API ключа

### Вариант 1: Через личный кабинет Mobiteam
1. Войдите в личный кабинет на https://gps.mobiteam.com.ua
2. Перейдите в раздел "Настройки" → "API"
3. Создайте новый API ключ или скопируйте существующий
4. Сохраните ключ в безопасном месте

### Вариант 2: Обращение в техподдержку
Если раздел API недоступен в вашем аккаунте:
1. Обратитесь в техническую поддержку Mobiteam
2. Запросите активацию доступа к API
3. Получите API ключ

## Настройка скрипта

### 1. Откройте файл `mobiteam_odometer.py`

### 2. Найдите секцию настроек (около строки 360):
```python
BASE_URL = "https://gps.mobiteam.com.ua"
API_KEY = "ваш_api_ключ"
```

### 3. Замените на свои данные:
- **BASE_URL** - обычно остается как есть
- **API_KEY** - ваш API ключ из личного кабинета

Пример:
```python
BASE_URL = "https://gps.mobiteam.com.ua"
API_KEY = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkw..."
```

## Как работает скрипт

Скрипт использует два метода получения данных:

### Метод 1: События геозоны (приоритетный)
1. Запрашивает события входа/выхода из геозоны за день
2. Находит первое событие "выезд" и последнее событие "въезд"
3. Получает показания одометра в момент этих событий

**Преимущества:**
- Быстрая работа
- Точные данные
- Минимальная нагрузка на API

### Метод 2: Анализ трека (резервный)
Если события геозоны недоступны:
1. Загружает весь трек движения за день
2. Анализирует каждую точку трека
3. Определяет моменты выхода/входа в геозону по координатам
4. Фиксирует показания одометра

**Преимущества:**
- Работает всегда
- Не зависит от настроек событий

## Особенности Mobiteam API

### Названия полей
API Mobiteam может возвращать данные с разными названиями полей:
- Одометр: `mileage`, `odometer`
- Широта: `latitude`, `lat`
- Долгота: `longitude`, `lon`
- Время: `timestamp`, `time`, `dt`

Скрипт учитывает все эти варианты.

### Типы геозон
Скрипт поддерживает **круговые геозоны** с использованием формулы Haversine для точного расчета расстояния.

### Ограничения API
- Рекомендуемый период запроса: до 31 дня
- Максимальное количество точек в треке: зависит от вашего тарифа
- Rate limit: уточните у техподдержки

## Возможные ошибки и решения

### Ошибка 401 (Unauthorized)
```
✗ Ошибка подключения: 401
```
**Решение:** 
- Проверьте правильность API ключа
- Убедитесь, что API ключ не истек
- Проверьте, что у вас есть права на использование API

### Ошибка 403 (Forbidden)
```
✗ Ошибка подключения: 403
```
**Решение:**
- API доступ не активирован в вашем аккаунте
- Обратитесь в техподдержку для активации

### Объекты не найдены
```
⚠ Объекты не найдены
```
**Решение:**
- Проверьте, что в вашем аккаунте есть добавленные транспортные средства
- Убедитесь, что API ключ имеет доступ к объектам

### Трек движения не найден
```
⚠ Трек движения не найден
```
**Решение:**
- Проверьте, была ли активность транспорта в выбранный день
- Убедитесь, что GPS терминал был включен и передавал данные
- Проверьте настройки хранения треков в вашем тарифе

### События геозоны недоступны
```
⚠ События геозоны не найдены, попробуем через трек
```
**Решение:**
- Это нормально, скрипт автоматически переключится на анализ трека
- Для использования событий убедитесь, что в настройках геозоны включена генерация событий

## Интеграция с путевым листом

Данные, полученные скриптом, можно использовать для автоматического заполнения путевого листа:

**Поля путевого листа:**
- Показание одометра при выезде → `first_exit['mileage']`
- Время выезда → `first_exit['time']`
- Показание одометра при возвращении → `last_enter['mileage']`
- Время возвращения → `last_enter['time']`
- Пробег за день → `last_enter['mileage'] - first_exit['mileage']`
